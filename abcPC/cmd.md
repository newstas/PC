# Командная строка (cmd.exe) Windows 10 

## Вызов командной строки
- **Быстрый запуск:** нажмите **Win+R**, введите `cmd` и нажмите **Enter**. 
- **С правами администратора**: нажмите правой кнопкой на **«Пуск» -> Командная строка (администратор)**.


## Полезные команды (шпаргалка)

- `cd \` — переход в корень диска.
- `dir` — просмотр списка файлов и папок в текущей директории.
- `ipconfig` — просмотр сетевых настроек и IP-адреса.
- `ping [адрес]` — проверка подключения к серверу/сайту.
- `sfc /scannow` — проверка и восстановление целостности системных файлов.
- `chkdsk` — проверка жесткого диска на ошибки.
- `help` — показать список доступных команд.


### Pathping - трассировка маршрута с оценкой качества каналов связи
- `pathping` - информацию о потере данных на промежуточных узлах
Pathping — это встроенная в Windows утилита командной строки для диагностики сети, объединяющая функции `ping` и `tracert`.
Она показывает маршрут до цели и вычисляет потерю пакетов на каждом промежуточном узле (маршрутизаторе). Это позволяет точно определить,
какой участок сети вызывает задержки или потерю данны на промежуточных узлах между начальным и конечным пунктом.


### netstat - проверка портов

- `netstat -ano` - проверяем занятые порты PC
- `netstat -ano | more` - проверяем занятые порты PC компактно

Ответ:
- «LISTENING» — port слушается;
- «ESTABLISHED» — соединение точка-точка установлено;
- «TIME WAIT» — превышение времени ответа;
- «CLOSE WAIT – соединение в ожидание закрытия.

Затем в Диспетчере задач находим PID (ИД процесса) и отрыв расположение файла, смотрим чем какой программой занят порт.

Отображает активные подключения TCP, порты, прослушиваемые компьютером, статистику Ethernet, таблицы маршрутизации IP, 
статистику IPv4 (для протоколов IP, ICMP, TCP и UDP) и IPv6 (для протоколов IPv6, ICMPv6, TCP через IPv6 и UDP через IPv6). 
Если запустить команду без параметров, то отобразятся только подключения TCP.


## Устройства, пробуждающие ПК — препятствуют переходу в спящий режим

**PowerCFG** — это мощная встроенная утилита командной строки Windows позволяющая управлять различными аспектами планов электропитания 
(также известных как схемы электропитания), диагностики энергопотребления, анализа батареи и настройки спящих режимов.
Выполнить команду в **cmd** или **PowerShell** (администратор).

### Список утройств, которым разрешено выводить систему из спящего режима

Вывести список всех устройств, которым разрешено выводить систему из спящего режима:

```
powercfg -devicequery wake_armed
```

\- откройте командную строку от имени администратора и введите команду, чтобы узнать, какое устройство мешает сну.
Мышь или сетевая карта часто вызывают это, но бывают и другие устройства, например:
```
C:\WINDOWS\system32>powercfg -devicequery wake_armed
NVIDIA USB Type-C Port Policy Controller // лишнее
HID-compliant Optical Wheel Mouse (001)
Logicool HID-Compliant Keyboard (106 keys)
Qualcomm Atheros AR8151 PCI-E Gigabit Ethernet Controller (NDIS 6.30) // лишнее
```
### Что мешает войти в режим Сна — каманда `-requests`

Посмотрите, что мешает войти в режим Сна:

```
powercfg -requests
```

Далее смотреть со статусом Выполнение. Если есть, то закрыть активность и проверить , что это ПО является причиной.

*Например:*
```
SYSTEM:
[DRIVER] USB Audio Device (USB\VID_1395&PID_0024&MI_00\8&51e117b&0&0000)
Аудиопоток уже используется.
[DRIVER] Устаревшая вызывающая сторона ядра // Legacy Kernel Caller
```
Для исправления это ситуации прибегаем к следующей команде `powercfg -requestsoverride`


### Команда игнорирования запросов питания `-requestsoverride`

Вы можете принудительно сказать Windows игнорировать этот драйвер при проверке состояния сна.

1. Введите следующую команду и нажмите Enter:
```
powercfg -requestsoverride DRIVER "USB Audio Device" SYSTEM
```  
*Если не помогло, попробуйте также:* 
```
powercfg -requestsoverride DRIVER "Legacy Kernel Caller" SYSTEM
```
2. Перезагрузите компьютер. 


**Просмотр:** Для просмотра текущих настроек переопределения введите:
```
powercfg -requestsoverride
```
*например появится:*
```
[SERVICE]

[PROCESS]

[DRIVER]
USB Audio Device SYSTEM
Legacy Kernel Caller SYSTEM
```
- **Удаление переопределения:** Чтобы отменить правило, оставьте список запросов пустым:
```
powercfg -requestsoverride  DRIVER "USB Audio Device"
```

Команда полезна, когда, например, драйвер сетевой карты (srvnet) или браузер не дают экрану погаснуть, хотя активных действий не происходит.
